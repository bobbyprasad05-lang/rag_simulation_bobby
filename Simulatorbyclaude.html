⚙️ Phase 1 — Indexing Prep Phase

Before an LLM can use external knowledge, we build a searchable knowledge base. Documents are split into chunks, converted into vector embeddings (numbers that capture meaning), and stored in a Vector Database. Similar concepts cluster close together in this vector space.

🔍 Phase 2 — Retrieval Semantic Search

When a user sends a query, it's converted into a vector embedding — just like the chunks. A semantic search finds the chunks whose vectors are closest to the query vector. This captures meaning, not just keyword matches.
